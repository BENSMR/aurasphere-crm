## ğŸš€ Implementation Details

### **Subscription Service**
```dart
class SubscriptionService {
  static final SubscriptionService _instance = SubscriptionService._internal();
  
  // Get current plan
  Future<String> getCurrentPlan(String orgId) async {
    final org = await supabase
        .from('organizations')
        .select('plan')
        .eq('id', orgId)
        .single();
    
    return org['plan'] as String;
  }

  // Check if plan is active
  Future<bool> isPlanActive(String orgId) async {
    final org = await supabase
        .from('organizations')
        .select('stripe_status, paddle_status, trial_ended_at')
        .eq('id', orgId)
        .single();
    
    final stripeActive = org['stripe_status'] == 'active';
    final paddleActive = org['paddle_status'] == 'active';
    final inTrial = org['trial_ended_at'] == null || 
                    DateTime.parse(org['trial_ended_at']).isAfter(DateTime.now());
    
    return stripeActive || paddleActive || inTrial;
  }

  // Upgrade plan
  Future<void> upgradePlan(String orgId, String newPlan) async {
    final currentPlan = await getCurrentPlan(orgId);
    
    // Create Stripe subscription
    final subscription = await StripePaymentService().createSubscription(
      orgId: orgId,
      newPlan: newPlan,
    );

    // Update database
    await supabase
        .from('organizations')
        .update({
          'plan': newPlan,
          'stripe_subscription_id': subscription['id'],
          'stripe_status': 'active',
        })
        .eq('id', orgId);

    // Log subscription event
    await supabase.from('subscriptions').insert({
      'org_id': orgId,
      'from_plan': currentPlan,
      'to_plan': newPlan,
      'event_type': 'upgraded',
      'amount': _getPlanPrice(newPlan),
      'effective_date': DateTime.now().toIso8601String(),
    });
  }

  // Downgrade plan
  Future<void> downgradePlan(String orgId, String newPlan) async {
    // Similar to upgrade
  }

  // Cancel subscription
  Future<void> cancelSubscription(String orgId, String reason) async {
    await StripePaymentService().cancelSubscription(orgId);

    await supabase
        .from('organizations')
        .update({
          'stripe_status': 'cancelled',
          'cancellation_requested_at': DateTime.now().toIso8601String(),
        })
        .eq('id', orgId);
  }

  // Get plan features
  Map<String, dynamic> getPlanFeatures(String plan) {
    return {
      'solo': {
        'team_members': 1,
        'mobile_devices': 2,
        'tablet_devices': 1,
        'features': ['dashboard', 'jobs', 'invoices', 'clients', 'expenses'],
      },
      'team': {
        'team_members': 3,
        'mobile_devices': 3,
        'tablet_devices': 2,
        'features': ['dashboard', 'jobs', 'invoices', 'clients', 'team', 'dispatch', 'payments'],
      },
      'workshop': {
        'team_members': 7,
        'mobile_devices': 5,
        'tablet_devices': 3,
        'features': ['all'],
      },
    }[plan] ?? {};
  }
}
```

## ğŸ¯ Upgrade/Downgrade Flow

### **Upgrade Scenario**
```
User on SOLO ($29/month) â†’ Upgrades to TEAM ($79/month)
1. Prorated billing: Calculate remaining days in current cycle
2. Charge: New plan price - Prorated credit from old plan
3. Immediate: Features updated, team member slots available
4. Recorded: Subscription event logged for analytics
```

### **Downgrade Scenario**
```
User on TEAM ($79/month) â†’ Downgrades to SOLO ($29/month)
1. Prorated billing: Calculate credit for remainder of cycle
2. Refund: Credit applied to next month or refunded
3. Effective: End of current billing period (safest approach)
4. Recorded: Subscription event logged
```

### **Cancellation Flow**
```
User requests cancellation
1. Confirm action (warn about data retention)
2. Record cancellation reason
3. Cancel subscription with payment processor
4. Effective: End of billing period
5. Data: Retained for 30 days in case of reactivation
6. Access: Removed at cancellation date
```
